Folder Items

  • Dependencies - Contains the framework and packages that are required for the application to run.
  • Pages - Details
    • Contains all the pages required for the project. Has two files:
  • Properties - Settings and definitions that are going to be used when running the app
  • wwwroot - Details
    • Contains all of the static files. Store HTML, CSS, JS, files here.
    • Cannot store C# or Razor files here.
  • appsettings.json - Loads appsettings.json in production, loads appsettings.developer.json in development
  • program.cs - Contains the entry point for the program. The Main Method is the first method which gets invoked.
  • startup.cs - Contains the startup class.

Creating & Building a Project

  • ASP.NET Core Web Application - Umbrella core application that can be expanded
  • Web Application - Uses Razor pages, can use MVC pages. Uses Razor pages as a default
  • Web Application (Model-View-Controller) - Uses MVC, can use Razor pages
  • Always check "Configure for HTTPS".
  • IIS Express - Internet Information Server.
  • RazorPagesUI - Uses Kestrel, a command line web server.
  • Individual Page Items
    • @page - Must be first line of file
      • Indicates that the file is a Razor page
    • @model ModelName - Connects cshtml file to cshtml.cs file
    • Page Layout
      • Used to have one set layout across all pages. Header, Footer, etc.
      • _ViewStart.cshtml in Pages is the default location

Writing C# within HTML file

  • C# expressions have to start with @
    • Ex: <h1> @DateTime.Now.ToShortDateString() </h1>
  • C# expressions with spaces have to be surrounded with parentheses
    • Ex: <p> Last week this time: @(DateTime.Now - TimeSpan.FromDays(7)) </p>
  • Can surround multi-line code with brackets - { }
  • If statement just needs one "@" at the IF. The else and else if do not need it.
  • Switch statements are the same as IF statements, just one @

Section Details

Pages Details

  • Path to different webpage based on the relation to the Pages folder
  • Ex: localhost/Forms/AddAddress
  • Add Page - Right click > Add > Razor Page
  • - Shared Folder:
    • _Layout.cshtml - The actual html page. Details
  • _ViewImports.cshtml- Pages to help to get started in Razor pages.
  • _ViewStart.cshtml- Pages to help to get started in Razor pages.
  • Index.cshtml - contains razor syntax, which is a combination of HTML markup and C#
    • Index.cshtml.cs - contains the logic and behaivor for each page
  • Three pages are template pages:
    • Error.cshtml
    • Index.cshtml - Website starting page. Details
    • Privacy.cshtml

Layout cshtml Details

  • Found in Pages > Shared > _Layout.cshtml
  • The line "@RenderBody()" injects the Razor page, the Index.cshtml page, into the HTML.
  • Can make changes to the overall HTML page here in _Layout.cshtml
  • Can add additional style sheet references to the page.

Index.cshtml Details

  • index.cshtml is the dotnet starting point
    • Industry standard is to call it Index
  • "@page" indicates it as a Razor page
    • Indicates the Razor syntax, C# code
  • "@model IndexModel" - the data that is associated with this page
    • PageModel is at Index.cshtml.cs
    • Will typically only have one model
    • Part of the PageModel
  • ViewData["Title"] = "Home page" - changes the Title property
  • Index.cshtml.cs Details:
    • C# class, has a constructor that takes in a logger
    • OnGet() Method - is what renders the Index page, is what runs when the Index page is created
    • OnPost() Method - runs whenever data is posted to the page
    • OnPost is not always included, only when information needs to be read in

wwwRoot Folder Details

  • Bootstrap Location: wwwroot > lib > bootstrap > dist > js > bootstrap.js
  • Uses Bootstrap version 4.xx
  • Should stay on version of Bootstrap initially provided.
